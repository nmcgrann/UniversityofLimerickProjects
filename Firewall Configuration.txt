ACLs I used
      I used a total of 5 ACLs.
	The border router contains the inbound ACL BR-IN on the interface connected to the internet. It blocks the Evil group, enforces anti-spoofing, and allows mail relay and HTTPS traffic from legitimate sources. The border router also contains the outbound ACL BR-OUT which handles outgoing traffic such as the mail relay SMTP and web server responses as well as chosen routing protocol. 
      The internal router contains the ACLS IR-IN-PUBLIC, IR-IN-SERVERS, WS-OUT and WS-IN (WS-OUT and WS-IN are reflexive, "together as one"). IR-IN-PUBLIC filters traffic from the public server network to the internal servers, controlling web server access to the database and mail relay access to the mail hub. The IR-IN-SERVERS regulates access to the internal database server and mail hub server from workstations and the partner network. The Workstation network used reflexive ACLs, WS-OUT and WS-IN applied to the internal router interface connected the workstations, which allow only outbound requests form the workstations and their corresponding return traffic, blocking unsolicited inbound connections.
